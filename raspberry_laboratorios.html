Prelaboratorio 1 - Instalacion S.O Raspbian

Esta leccion tiene como objetivo introducir y briandarles las herramientas para la instalacion del sistema operativo Raspbian, para los equipos raspberry PI 3B que se encuentran en la universidad. Antes de iniciar es necesario tener una memoria micro SD, como minimo de 32GB y preferiblemente de clase 10, esto para evitar conflictos posteriores con el hardware de sistema embebido. A continuacion se muestra el paso a paso para la descarga, instalacion y puesta en marcha del sistem operativo Raspbian

  1. Ingresar al siguiente link Raspberry Pi OS y descargar la version de sistema operativo correspondiente a la version de raspberry. En este caso, Raspberry PI OS with desktop
  2. Luego, ingresa al siguiente link Balena , y descarga el programa
  3. Extre el archivo ZIP que contiene el sistema operativo Raspbian
  4. Ejecuta como administrador el programa Balena decargado, agrega el archivo iso descomprimido, selecciona la Micro SD conectada al PC e instala el sistema operativo en la Micro SD
  5. Inserta la micro SD en el socket correspondiente de la raspberry y conecta los perifericos (Mause, teclado y monitor)
  6. Conecta la raspberry a la fuente de energia, espera un momento y configura las opciones basicas de instalacion

Laboratorio 1 - Encer y apagar LED con interfaz grafica

Probar y corregir el siguiente codigo

Este código usa la biblioteca Tkinter para crear una interfaz gráfica y RPi.GPIO para controlar los pines GPIO de una Raspberry Pi. Primero, se importan las bibliotecas necesarias, Tkinter para la interfaz gráfica y RPi.GPIO para interactuar con los pines GPIO. Luego, se configura el pin GPIO 18 en modo salida mediante GPIO.setup(18, GPIO.OUT, pull_up_down=GPIO.PUD_UP, initial=1), donde el modo BCM establece que la numeración de los pines sigue el esquema del chip Broadcom, GPIO.OUT define que el pin será usado como salida y PUD_UP activa una resistencia pull-up interna para garantizar que el pin no quede en un estado indefinido. La ventana principal de la interfaz se crea usando root = Tk(), y se le asigna un título ("GPIO Test") y un color de fondo (#FFFFF0). El código define dos funciones, onBtnPress() y offBtnPress(), que son llamadas al presionar los botones "On" y "Off", respectivamente. En la función onBtnPress(), si la variable debug es True, se imprime un mensaje de depuración indicando que el botón "On" fue presionado, y se cambia el estado del pin GPIO 18 a alto (GPIO.output(18, 1)), encendiendo así el dispositivo conectado. Similarmente, offBtnPress() apaga el dispositivo poniendo el pin en bajo (GPIO.output(18, 0)). Para organizar los botones en la ventana, se crea un Frame llamado primaryFrame dentro de la ventana principal root. El frame se posiciona usando el método grid(), que organiza los elementos en una cuadrícula. En este caso, se coloca en la fila 0 y columna 0, con un espacio adicional (padx=10, pady=10) para separar los elementos del borde. Luego, se crean dos botones, uno para encender el dispositivo (onBtn) y otro para apagarlo (offBtn), usando el widget Button(). Cada botón está asociado a una función (usando command=onBtnPress y command=offBtnPress) y se posicionan en el frame usando el método grid(), uno debajo del otro. Finalmente, se ejecuta root.mainloop(), que es el bucle principal de Tkinter que mantiene la ventana abierta y activa, esperando la interacción del usuario. Este código proporciona una interfaz gráfica simple para controlar un dispositivo conectado a un pin GPIO de una Raspberry Pi, usando botones para encenderlo y apagarlo, mientras que se gestionan los estados de los pines mediante la biblioteca RPi.GPIO.
